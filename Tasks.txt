MVP фичи:

Отели:
- Создание, изменение данных об отеле
- Удаление отеля только если нету броней
- Включение/Отключение отеля для приема броней
- Получение списка отелей всех (айди, название, кол-во звезд, долготу, широту вернуть для каждого отеля сети)

Категории номеров:
- Создание, изменение данных о номере для отеля
- Добавление/обновление/удаление опций для номера
- Добавление/обновление/удаление удобств для номера
- Добавление/обновление/удаление тарифа для номера
- Добавление/обновление/удаление сервисов для тарифа

Опции которые могут юзать номера как доп штуки по номеру (доп кровать):
- CRUD
При удалении проверять есть ли связь с каким либо номером, если есть - не удалять

Категории удобств:
- CRUD
При удалении проверять есть ли связь с каким либо удобством, если есть - не удалять
  
Удобства:
- CRUD
При удалении проверять на наличие удобства в каком либо номере, если есть - не удалять

Сервисы от отеля для гостей:
- CRUD
При удалении проверять на использование, если есть - не удалять


Изменять базовую цену номера, сервисов, предпочтений МОЖНО, но только раз в неделю (независимо от того принимает отель брони или нет)

Обработка ошибок и исключений:
Для фронтенда ошибки возвращать в формате problem+json, каждая ошибка имеет код в таком формате:
hotel.description.tooLong
hotel.notFound

Ошибка содержит описание по русски написанное и может какие то метаданные (айди отеля например)

Если ошибка по вине сервера то вернуть 500 ошибку без описания
